## 一、项目概述

RuView 是一个**基于 WiFi 信号的边缘 AI 感知系统**，核心能力是通过分析 WiFi 信号中的 CSI（Channel State Information，信道状态信息）来实现：

- **人体姿态估计** — 无需摄像头，通过 WiFi 信号重建人体姿态
- **生命体征检测** — 呼吸频率（6-30 次/分钟）、心率（40-120 bpm）
- **存在感知** — 判断房间内是否有人，延迟 < 1ms
- **穿墙感知** — WiFi 信号可穿透墙壁，深度可达 5 米
- **跌倒检测** — 检测到跌倒后 < 2 秒发出警报

**核心理念**：不依赖摄像头、不需要穿戴设备、不需要联网，纯粹利用物理信号（WiFi 电磁波）实现空间感知。

---

## 二、技术原理（简述）

```
WiFi 路由器发射无线电波 → 电波穿过房间 → 碰到人体后散射
    ↓
ESP32 传感器网格（4-6 个节点）在多个频道上捕获 CSI 数据
    ↓
多频段融合：3 个频道 × 56 个子载波 = 168 个虚拟子载波
    ↓
多站融合：N×(N-1) 条链路 → 注意力加权的跨视角嵌入
    ↓
信号处理：Hampel 滤波、SpotFi、菲涅尔、BVP、频谱图 → 干净特征
    ↓
AI 骨干网络 (RuVector)：注意力机制、图算法、压缩、场模型
    ↓
神经网络推理 → 17 个身体关键点 + 生命体征 + 房间模型
    ↓
输出：实时姿态、呼吸、心率、房间指纹、漂移警报
```

**关键概念**：
- **CSI（信道状态信息）**：WiFi 信号在每个子载波上的幅度和相位数据，比普通的 RSSI（信号强度）精细得多
- **子载波**：WiFi 信号被分成多个窄频子通道，每个子载波独立受环境影响
- **多站式**：多个传感器节点之间两两形成信号路径，可以从多个角度"看"到目标

### 3.4 硬件支持

项目原生支持的硬件平台：
- **ESP32-S3 传感器网格**（推荐）：3-6 个节点，每个约 $8，支持 CSI 采集
- **研究级网卡**：Intel 5300 / Atheros AR9580
- **普通 WiFi**：仅支持 RSSI 级别的粗略存在检测

### 3.5 应用场景（与养老院直接相关）

项目 README 中已明确列出的养老院场景：
> **Elderly care / assisted living** — 跌倒检测、夜间活动监控、睡眠呼吸监测 — 不需要穿戴设备。硬件需求：每房间 1 个 ESP32-S3（$8），跌倒警报 < 2 秒。

## 六、总结

RuView 是一个**成熟度较高**的 WiFi 感知项目，具备：
1. 完整的信号处理管线（从原始 CSI 到姿态输出）
2. 生产级 Rust 实现（1300+ 测试，54K fps）
3. 明确的养老院应用场景支持
4. 灵活的部署选项（边缘设备/服务器/浏览器）
5. 隐私优先设计（无摄像头，无视频数据）

**对你的养老院项目来说，这个项目在技术上完全匹配需求。关键问题在于你的硬件设备（Cudy TR3000）能否替代 ESP32 完成 CSI 数据采集。** 详见第二份文档。
